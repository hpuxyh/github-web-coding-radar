四个榜单分别看什么

先从 GitHub 官方数据和 README 里找候选项目，再算四类分数。同一个项目只放进最合适的一个榜，避免重复。

热度榜

看最近涨得快不快、是不是刚出现、最近有没有更新，以及是否踩中 AI 编程、Agent、MCP 等热门方向。

大家都在用

看 fork、README、截图示例、官网和持续维护情况，更偏向已经有人拿去试用、改造或集成的项目。

高收藏

看累计 stars 和 forks，找长期被开发者认可、适合补课和对标的项目。

参与讨论

看 issue、fork、最近更新和公开协作痕迹，找有人反馈、有人参与、社区还在推动的项目。

冲突时按热度榜、大家都在用、参与讨论、高收藏的顺序归类。排序下拉框只调整当前榜单顺序，不会重新分组。
